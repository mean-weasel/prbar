Small surface, useful signal.

PR Menu Bar keeps the activity view close to the work without becoming another dashboard to maintain.

GitHub-backed refresh

Fetch merged pull requests across repositories and keep the last useful activity visible between refreshes.

Repository breakdown

Scan the weekly shape of work, then drop into the repositories that are carrying most of the movement.

Failure-aware updates

Refresh errors do not wipe the view. Rate-limit and token issues are surfaced without hiding the previous data.

Works as a prototype, stays honest about data.

Use sample mode to explore the app. Add a GitHub token in the launch environment when you want live repository activity.

  • Live data when you provide a token
  • Sample mode when you do not
# run with live GitHub data
$ PR_MENU_BAR_GITHUB_TOKEN=github_pat_xxx make run

# run local checks
$ make ci-local
